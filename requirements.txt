# Minimum-version pin, not exact - psycopg2-binary 2.9.9 predates Python
# 3.13's release and has no ready-made installer for it; 2.9.10+ does.

# These three use minimum-version pins (>=), not exact pins (==), on
# purpose. They contain compiled C code, and if pip can't find a
# ready-made installer (a "wheel") for your exact Python version, it
# tries to compile the package from raw source instead - which needs a
# full C/C++ toolchain and fails with a wall of Meson/Ninja/compiler
# errors (exactly what you'd see if this happens). Using ">=" instead of
# "==" lets pip fetch whatever is the newest available version instead of
# insisting on an old one that might predate your Python version's wheel
# support. pandas 2.2.2, for example, was one patch release too old for
# Python 3.13 - 2.2.3 was the first version with a 3.13 installer.
numpy>=2.1.3
pandas>=2.2.3
scikit-learn>=1.5.2
